你是一位精通 Vue 3 + Express + MySQL 全栈开发的高级工程师。你的任务是完善 GPUStore 电商项目的订单购买与支付流程，实现以下业务逻辑：

1. 商品详情页点击"立即购买" → 生成待付款订单 → 进入结算中心
2. 购物车点击"提交订单" → 生成待付款订单 → 进入结算中心
3. 结算中心点击"立即结算" → 支付成功 → 订单进入"待收货" → 30秒后自动变为"待评价"
4. 结算中心未支付直接退出 → 订单保持"待付款"，可在订单列表继续支付

---

### 一、核心流程与状态定义

订单状态流转：
- `pending`（待付款）：创建订单后的初始状态
- `shipped`（待收货）：用户点击"立即结算"支付成功后
- `delivered`（待评价）：支付成功页倒计时 30 秒后自动流转
- `completed`（已完成）：用户点击"去评价"后（评价功能可后续再做）
- `cancelled`（已取消）：用户主动取消

```javascript
  const { id } = req.params;
  const userId = req.user.id;
  try {
    const [orders] = await db.execute(
      'SELECT * FROM orders WHERE id = ? AND user_id = ?',
      [id, userId]
    );
    if (orders.length === 0) {
      return res.status(404).json({ message: '订单不存在' });
    }
    const order = orders[0];
    const [items] = await db.execute(
      'SELECT * FROM order_items WHERE order_id = ?',
      [order.id]
    );
    order.items = items;
    res.json(order);
  } catch (err) {
    console.error('获取订单详情失败:', err);
    res.status(500).json({ message: '获取订单详情失败' });
  }
});
```

```javascript
async onSubmit() {
  if (this.submitting) return;
  
  // 如果有 orderId，走"支付已有订单"流程
  if (this.orderId) {
    this.submitting = true;
    try {
      // 调用后端修改订单状态为 shipped（待收货）
      await orderAPI.updateOrderStatus(this.orderId, 'shipped');
      // 跳转到支付成功页，携带订单 ID
      this.$router.push({ path: '/paysuccess', query: { orderId: this.orderId } });
    } catch (error) {
      const msg = error.response?.data?.message || '支付失败，请稍后重试';
      showToast(msg);
    } finally {
      this.submitting = false;
    }
    return;
  }

  // 如果没有 orderId，走原来的"从购物车创建订单"兼容逻辑
  const payProds = this.carts.filter(item => item.checked);
  if (payProds.length === 0 || this.sumPrice <= 0) {
    showToast('请选择要购买的商品');
    return;
  }
  if (!this.currentAddress || !this.currentAddress.id) {
    showToast('请先选择收货地址');
    return;
  }

  const insufficientItems = payProds.filter(item => item.stock !== undefined && item.num > item.stock);
  if (insufficientItems.length > 0) {
    const names = insufficientItems.map(i => i.title).join('、');
    showToast(`${names} 库存不足，请调整数量`);
    return;
  }

  this.submitting = true;
  const store = shopStore();
  try {
    await store.createOrderAPI({
      totalPrice: this.sumPrice,
      items: payProds.map(item => ({
        product_id: item.product_id,
        title: item.title,
        price: item.price,
        quantity: item.num,
        image_url: item.image || item.img || ''
      }))
    });
    this.$router.push("/paysuccess");
  } catch (error) {
    const msg = error.response?.data?.message || '订单提交失败，请稍后重试';
    showToast(msg);
  } finally {
    this.submitting = false;
  }
}
```

```vue
<template>
  <div class="success">
    <van-icon name="checked" />
    <h3>支付成功</h3>
    <p v-if="!delivered">
      您的商品正在发货中，预计 <span class="countdown">{{ countdown }}秒</span> 后送达
    </p>
    <p v-else>
      商品已送达！请前往

      进行评价
    </p>
    <p class="links">

      &nbsp;|&nbsp;
      <router-link to="/home">返回首页</router-link>

  </div>

<script>
import { orderAPI } from '@/services/api';
import { showToast } from 'vant';

export default {
  name: "PaySuccess",
  data() {
    return {
      orderId: null,
      countdown: 30,
      delivered: false,
      timer: null
    }
  },
  async mounted() {
    this.orderId = this.$route.query.orderId || null;
    if (this.orderId) {
      this.startDeliveryTimer();
    }
  },
  beforeUnmount() {
    if (this.timer) {
      clearInterval(this.timer);
    }
  },
  methods: {
    startDeliveryTimer() {
      this.timer = setInterval(async () => {
        this.countdown--;
        if (this.countdown <= 0) {
          clearInterval(this.timer);
          this.timer = null;
          await this.autoDeliver();
        }
      }, 1000);
    },
    async autoDeliver() {
      if (!this.orderId) return;
      try {
        await orderAPI.updateOrderStatus(this.orderId, 'delivered');
        this.delivered = true;
        showToast('商品已送达');
      } catch (error) {
        console.error('自动流转订单状态失败:', error);
      }
    }
  }
}
</script>
```

### 三、修改原则

1. **兼容旧流程**：结算中心 `RtIndex.vue` 仍然支持不带 `orderId` 直接从购物车进入的兼容模式。
2. **先创建后支付**："立即购买"和"提交订单"都必须先调用后端创建 `pending` 订单，再跳转到结算中心。
3. **状态校验**：后端 `PUT /orders/:id/status` 已有状态机校验，前端调用时无需额外校验。
4. **错误处理**：所有异步操作必须有 `try-catch`，失败时给用户 `showToast` 提示。
5. **定时器清理**：`RtPaySuccess.vue` 的倒计时定时器必须在 `beforeUnmount` 中清除，防止内存泄漏。
6. **不要修改**与本次任务无关的代码。
