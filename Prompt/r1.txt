你是一位精通 Vue 3 + Express + MySQL 全栈开发的高级工程师。你的任务是修复和完善的 GPUStore 电商项目的订单模块，解决订单同步失败、库存扣减不一致、订单状态流转缺失等问题。

---

### 一、核心问题诊断（你必须了解）

1. **订单同步失败根因**：
   - `src/store/shopStore.js` 中的 `createOrderAPI` 方法**先**创建本地假订单（id = Date.now()）、清空购物车，**后**异步调用后端 `orderAPI.createOrder`。
   - 后端返回的真实数据库 `orderId` 与前端本地 `Date.now()` 不一致，导致 `RtOrderList.vue` 的 `loadOrders` 去重逻辑失效，订单列表出现重复。
   - 如果后端创建失败（如库存不足），前端购物车已被清空，订单也未真正创建，用户无感知，数据严重不一致。

2. **库存扣减逻辑缺陷**：
   - `backend/routes/orders.js` 第 35 行查询商品时使用了 `SELECT id, name, price, stock`，但 `products` 表中商品名字段是 `title`，不存在 `name` 字段。
   - 创建订单时库存已扣减，但**取消订单时库存没有回滚**。
   - 前端在调用后端前就清空了购物车，后端失败无法恢复。

3. **订单状态流转不完整**：
   - 数据库 `orders.status` 当前枚举为：`pending`（待付款）、`paid`（已付款）、`shipped`（已发货）、`delivered`（已送达）。
   - 缺少 `cancelled`（已取消）、`completed`（已完成/待评价）状态。
   - 前端订单列表 `RtOrderList.vue` 有"待付款"、"待收货"、"待评价"三个空 tab，没有任何过滤逻辑和操作按钮。
   - 后端缺少：取消订单、修改订单状态、删除订单的接口。

#### 2.1 修复 `src/store/shopStore.js` 的 `createOrderAPI`

将现有逻辑改为**先调后端，成功后更新前端**的可靠流程：

```javascript
async createOrderAPI(orderData) {
  const store = this;
  const checkedCarts = this.getCheckedCarts();
  
  // 1. 先调用后端创建订单
  let backendOrder;
  try {
    backendOrder = await orderAPI.createOrder({
      items: (orderData.items || []).map(item => ({
        product_id: item.product_id,
        quantity: item.quantity || item.num || 1
      }))
    });
  } catch (error) {
    // 后端失败：抛出错误，让调用方处理，购物车不动
    console.error('后端创建订单失败:', error);
    throw error;
  }

  // 2. 后端成功：清空购物车（删除后端购物车项 + 更新本地状态）
  const checkedIds = checkedCarts.map(c => c.id);
  for (const cart of checkedCarts) {
    try {
      await cartAPI.removeFromCart(cart.id);
    } catch (error) {
      console.warn('删除购物车项失败:', error.message);
    }
  }
  store.carts = store.carts.filter(c => !checkedIds.includes(c.id));

  // 3. 用后端返回的真实订单数据更新本地 store
  const realOrder = {
    id: backendOrder.id,
    user_id: backendOrder.user_id,
    total_price: backendOrder.total_price,
    status: backendOrder.status,
    created_at: backendOrder.created_at,
    items: (backendOrder.items || []).map(item => ({
      product_id: item.product_id,
      product_name: item.product_name,
      price: item.price,
      quantity: item.quantity,
      image_url: item.image_url
    }))
  };
  store.orders = [realOrder, ...store.orders];
  return realOrder;
}
```

**关键点**：不要在前端自己生成 `Date.now()` 作为订单 ID。所有订单数据必须以**后端数据库真实数据**为准。

**B. 添加取消订单接口（回滚库存）**

```javascript
// 取消订单
router.put('/:id/cancel', async (req, res) => {
  const { id } = req.params;
  const userId = req.user.id;
  const connection = await db.getConnection();
  try {
    await connection.beginTransaction();
    
    // 校验订单归属和状态
    const [orders] = await connection.execute(
      'SELECT * FROM orders WHERE id = ? AND user_id = ?',
      [id, userId]
    );
    if (orders.length === 0) {
      await connection.rollback();
      return res.status(404).json({ message: '订单不存在' });
    }
    const order = orders[0];
    if (order.status !== 'pending') {
      await connection.rollback();
      return res.status(400).json({ message: '只能取消待付款订单' });
    }

    // 回滚库存
    const [items] = await connection.execute(
      'SELECT * FROM order_items WHERE order_id = ?',
      [id]
    );
    for (const item of items) {
      await connection.execute(
        'UPDATE products SET stock = stock + ? WHERE id = ?',
        [item.quantity, item.product_id]
      );
      await connection.execute(
        'UPDATE product_summary SET stock = stock + ? WHERE id = ?',
        [item.quantity, item.product_id]
      );
    }

    // 更新订单状态为已取消
    await connection.execute(
      "UPDATE orders SET status = 'cancelled' WHERE id = ?",
      [id]
    );

    await connection.commit();
    res.json({ message: '订单已取消', id: parseInt(id) });
  } catch (err) {
    await connection.rollback();
    console.error('取消订单失败:', err);
    res.status(500).json({ message: '取消订单失败' });
  } finally {
    connection.release();
  }
});
```

**C. 添加修改订单状态接口**

```javascript
// 修改订单状态（支持：pending -> paid -> shipped -> delivered -> completed）
router.put('/:id/status', async (req, res) => {
  const { id } = req.params;
  const { status } = req.body;
  const userId = req.user.id;

  const validTransitions = {
    'pending': ['paid', 'cancelled'],
    'paid': ['shipped'],
    'shipped': ['delivered'],
    'delivered': ['completed']
  };

  if (!validTransitions[status] && status !== 'cancelled') {
    return res.status(400).json({ message: '无效的目标状态' });
  }

  try {
    const [orders] = await db.execute(
      'SELECT * FROM orders WHERE id = ? AND user_id = ?',
      [id, userId]
    );
    if (orders.length === 0) {
      return res.status(404).json({ message: '订单不存在' });
    }

    const currentStatus = orders[0].status;
    const allowedNext = validTransitions[currentStatus] || [];
    if (!allowedNext.includes(status)) {
      return res.status(400).json({ message: `订单当前状态为 ${currentStatus}，无法变更为 ${status}` });
    }

    await db.execute('UPDATE orders SET status = ? WHERE id = ?', [status, id]);
    res.json({ message: '状态更新成功', id: parseInt(id), status });
  } catch (err) {
    console.error('更新订单状态失败:', err);
    res.status(500).json({ message: '更新订单状态失败' });
  }
});
```

**D. 添加删除订单接口**

```javascript
// 删除订单（仅允许删除已完成或已取消的订单）
router.delete('/:id', async (req, res) => {
  const { id } = req.params;
  const userId = req.user.id;
  const connection = await db.getConnection();
  try {
    await connection.beginTransaction();

    const [orders] = await connection.execute(
      'SELECT * FROM orders WHERE id = ? AND user_id = ?',
      [id, userId]
    );
    if (orders.length === 0) {
      await connection.rollback();
      return res.status(404).json({ message: '订单不存在' });
    }
    if (!['completed', 'cancelled'].includes(orders[0].status)) {
      await connection.rollback();
      return res.status(400).json({ message: '只能删除已完成或已取消的订单' });
    }

    await connection.execute('DELETE FROM order_items WHERE order_id = ?', [id]);
    await connection.execute('DELETE FROM orders WHERE id = ?', [id]);

    await connection.commit();
    res.json({ message: '订单已删除', id: parseInt(id) });
  } catch (err) {
    await connection.rollback();
    console.error('删除订单失败:', err);
    res.status(500).json({ message: '删除订单失败' });
  } finally {
    connection.release();
  }
});
```

  // 获取订单
  getOrders: () => api.get('/orders').then(res => res.data),

### 三、修改原则与约束

1. **数据一致性优先**：前端订单数据必须以后端数据库为准，禁止前端自行生成订单 ID。
2. **事务安全**：所有涉及库存回滚的操作必须使用 `db.getConnection()` + `beginTransaction()` + `commit()/rollback()`。
3. **最小改动**：不要重构整个项目，只修改与订单相关的文件和逻辑。
4. **错误处理**：所有异步操作必须有 `try-catch`，失败时给用户明确的 `showToast` 提示。
5. **状态校验**：后端修改订单状态时必须校验当前状态是否允许目标状态的流转，防止非法跳转。
6. **权限校验**：订单操作必须校验 `user_id`，确保用户只能操作自己的订单。
7. **不要修改**与本次任务无关的代码（如商品列表、收藏、地址等模块）。
